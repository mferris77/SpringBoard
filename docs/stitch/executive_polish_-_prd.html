Here is the updated Product Requirements Document for the "Executive Polish" design package, tailored for Kantar Efficiency & Automation. # Executive Polish ## Product Overview **The Pitch:** "Kantar Efficiency & Automation" (North America) is a high-visibility showcase portal for the NA Operations group. It aggregates "skunkworks" internal tools, data processing automations, and efficiency metrics into a prestigious digital dashboard that validates the team's impact on market research workflows. **For:** Regional Leadership (to validate innovation ROI) and Client Service / Project Managers (direct users of the automation tools). **Device:** Desktop (Primary), Tablet (Secondary) **Design Direction:** "Corporate Noir." A stark, authoritative aesthetic utilizing high-contrast black and white to convey clarity and precision. It feels expensive, final, and trustworthy. **Inspired by:** *The New York Times Magazine*, *McKinsey Quarterly*, *Pentagram* case studies. --- ## Screens - **01 Executive Dashboard:** The landing page. High-level metric aggregation and "hero" innovation highlights. - **02 Project Index:** A searchable, filterable grid of all completed data initiatives and automation case studies. - **03 Case Study Detail:** In-depth narrative of a specific project, focusing on "Before/After" processing speeds. - **04 Tool Launcher:** A functional gateway for active internal SaaS tools used by Client Service. - **05 Request Portal:** A structured intake form for new "out of the box" automation requests. --- ## Key Flows **Flow: Leadership Review** 1. User lands on **01 Executive Dashboard** -> Absorbs top-level "Hours Saved Year to Date". 2. Clicks "View 2024 Innovations" -> Navigates to **02 Project Index**. 3. Filters by "High Impact" -> Selects "Survey Data Cleaning Bot". 4. Lands on **03 Case Study Detail** -> Reviews ROI charts and downloads the one-pager PDF. **Flow: Project Manager Tool Access** 1. User lands on **01 Executive Dashboard**. 2. Clicks "Launchpad" in global nav -> Navigates to **04 Tool Launcher**. 3. Identifies "Verbatim Coder" via color-coded tag. 4. Clicks "Launch" -> External service opens in new tab. ---
Design System ## Color Palette The palette is strictly monochromatic with strategic metallic accents. - **Primary:** `#000000` - Backgrounds, primary text, heavy borders. - **Background:** `#FFFFFF` - Card surfaces, content areas. - **Surface:** `#F4F4F4` - Subtle section delineation (warm gray). - **Text:** `#111111` - Body text (almost black, never grey). - **Muted:** `#9CA3AF` - Meta data, inactive states. - **Accent:** `#D4AF37` - (Metallic Gold) ROI metrics, primary CTAs, key lines. - **Functional Red:** `#991B1B` - Status: Critical / Offline. - **Functional Blue:** `#1E3A8A` - Category: Finance / Legal. ## Typography Contrast between "Old World" authority and "New World" precision. - **Headings:** *Playfair Display*, 700 (Bold). Sharp serifs, high contrast. - **Sub-Headings:** *Playfair Display*, 400 (Italic). For distinctive emphasis. - **Body:** *Switzer* (or *Inter* tight), 400. Geometric, clean, unobtrusive. - **Data/Mono:** *JetBrains Mono*, 500. Used exclusively for raw numbers and code snippets. **Hierarchy:** - **H1:** 64px, -2% tracking, Playfair Display Bold. - **H2:** 42px, -1% tracking, Playfair Display Regular. - **H3:** 24px, 0% tracking, Playfair Display Italic. - **Body:** 16px, 150% line-height, Switzer. - **Metric:** 56px, JetBrains Mono, Gold. **Style Notes:** - **Borders:** Heavy use of 1px and 2px solid black borders. No soft shadows. - **Grid:** Visible grid lines are encouraged. - **Corners:** Sharp (0px radius) everywhere. No rounding. ## Design Tokens ```css :root { --color-black: #000000; --color-white: #FFFFFF; --color-gold: #D4AF37; --color-surface-grey: #F4F4F4; --font-serif: 'Playfair Display', serif; --font-sans: 'Switzer', sans-serif; --font-mono: 'JetBrains Mono', monospace; --border-thin: 1px solid #000000; --border-thick: 3px solid #000000; --spacing-unit: 24px; /* Rigid grid base */ } ```

Screen Specifications ### 01 Executive Dashboard **Purpose:** Immediate validation of the NA Operations team's value. The "Shop Window." **Layout:** - **Top:** Sticky minimalist header (Logo left, Links right). - **Hero:** Split screen. Left: Massive serif headline. Right: Live ticker of total hours saved. - **Middle:** 3-column grid of "Featured Wins" (Cards). - **Bottom:** "Manifesto" text block. **Key Elements:** - **Global Header:** 80px height, white bg, 1px bottom border. Links: Index, Launchpad, Request. - **Impact Ticker:** Background `#000000`. Text `#D4AF37`. Displays "14,203 Hours Returned to Business" in Mono font. - **Featured Card:** White bg, 1px black border. - **Top:** 200px grayscale image (dithered effect). - **Middle:** H3 Title "Survey Logic Automation". - **Bottom:** Gold metric "+400% Speed". **States:** - **Loading:** Skeleton screens with shimmering gold pulse. **Interactions:** - **Hover Card:** Image scales 105% (grayscale -> color), border thickness increases to 3px. **Responsive:** - **Desktop:** 3-column grid. - **Tablet:** 2-column grid. - **Mobile:** Single column stack. Ticker moves to top. ### 02 Project Index **Purpose:** Archival proof of work. Browse all "skunkworks" initiatives. **Layout:** - **Top:** Filter bar (Department, Status, Year) - styling resembles a technical spec sheet. - **Body:** Masonry grid of project cards. **Key Elements:** - **Filter Bar:** Sticky. 60px height. Dropdowns are text-only with chevron. Active filter gets Gold underline. - **Project Card (Standard):** - **Header:** Project Code (e.g., KAN-NA-24) in Mono. - **Title:** "Cross-Tab Generator". - **Tags:** Pill shape, white bg, black border. - **Metric:** "2d reduction" in Gold. **States:** - **Empty Filter:** "No initiatives match criteria." Centered, serif italic. **Components:** - **Filter Dropdown:** White bg, sharp corners, hover item `#F4F4F4`. ### 03 Case Study Detail **Purpose:** Educational deep dive. Explains *how* the data processing efficiency was achieved. **Layout:** Editorial style. Single column text, wide margins, floating data visualizations. **Key Elements:** - **Header:** Full width Title. "The Global Logistics Reset." - **Summary Box:** Sidebar (sticky on desktop). Contains: Team Members, Duration, Tech Stack. - **The "Before" State:** Grey background block (`#F4F4F4`). Serif text describing the manual pain points. - **The "Solution" State:** White background. Sans-serif text describing the automation. - **Metric Visualizer:** Bar chart using only Black and Gold bars. **Interactions:** - **Scroll:** "Summary Box" sticks to right edge while reading. ### 04 Tool Launcher **Purpose:** Utility. The daily driver for Client Service needing to use the tools. **Layout:** Dense Grid. High information density. Less "marketing," more "control panel." **Key Elements:** - **Tool Row:** List view (not card view). - **Col 1:** Icon (Stroke only, black). - **Col 2:** Tool Name (Bold Sans). - **Col 3:** Status Indicator (Green dot = Online, Red dot = Maint). - **Col 4:** "Launch" Button. - **Launch Button:** Solid Black rect, White text "OPEN >", Gold border on hover. **Responsive:** - **Mobile:** Converts from Table Row to Card view. ### 05 Request Portal **Purpose:** Intake. Structured form for business units to request "out of the box" solutions. **Layout:** Centered single column. Very form-focused. **Key Elements:** - **Input Fields:** No background, only bottom border (2px black). Label floats above. - **Category Select:** Large clickable tiles (Data Processing, Scripting, Reporting). Selected state: Black background, White text. - **Submit Button:** Full width, Gold background, Black text, Bold Serif.

Build Guide **Stack:** HTML + Tailwind CSS v3 **Build Order:** 1. **01 Executive Dashboard:** Defines the grid, typography scale, and the crucial "Card" component which is reused everywhere. 2. **03 Case Study Detail:** Establishes the editorial typography and complex layout patterns. 3. **02 Project Index:** Implements filtering logic based on the cards built in step 1. 4. **04 Tool Launcher:** Introduces the dense list-view/table components. 5. **05 Request Portal:** Form elements and input states. **Tailwind Config Highlights:** - Extend `colors` with `gold: '#D4AF37'`. - Extend `fontFamily` with `serif: ['Playfair Display']`. - Set `borderRadius: { DEFAULT: '0px' }`.
